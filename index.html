Controversial: Separate Meetings

Other interesting ideas

  • Request presentations
  • Detailed people page for the MCIN site
  • Presentations don't need to be formal
  • Scientific/invited talks related to tools or development
  • Social events outside the meeting (e.g. Friday beers)
  • Meeting shouldn't be after lunch (too sleepy to focus?)

Proposed Meeting Format

  • 0-15 minutes: Adminstrative updates from managers and project leads
    • Only report if they feel it's relevant to the group
  • 0-20 minutes: Moderated open discussion
    • Anyone can openly discuss what they've been working on, if they choose to
    • Ask for help with problems
    • Request presentations
  • 10-15 minutes: Presentation
    • Doesn't need to be formal
    • Should be related to software development, but not necessarily programming
    • Other potential topics could be sys admin, science, writing, design, etc.
    • Optional: anyone who isn't interested can leave at this point
  • 5-10 minutes: Questions and discussion
